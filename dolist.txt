项目需求：Bilibili 评论舆情多维对比分析平台
1. 项目背景

开发一个基于 Python 后端和 pyecharts 前端的评论分析工具。模仿 Google Trends 的交互风格，实现多个 BV 号视频评论数据的深度对比、热度演变及用户画像分析。
2. 技术栈要求

    后端：Python (Flask/FastAPI), Pandas (数据处理), Jieba (中文分词)。

    前端：HTML5, CSS Grid (响应式布局), ECharts.js (可视化)。

    存储：CSV 文件存储（字段包含：rpid, mid, user, level, location, comment, likes, replies, time, root, parent）。

3. 核心功能模块
A. 动态对比管理（Header/Comparison Bar）

    支持输入多个 BV 号并动态生成“标签（Tags）”。

    每个 BV 号分配唯一色值，并在全局图表中统一颜色。

    支持实时爬取新 BV 号或从本地历史记录中加载。

B. 时间轴热度对比（Trend Analysis）

    核心逻辑：将绝对时间转换为相对时间（如：发布后第 N 小时/天），使不同发布日期的视频具有可比性。

    展示：多线折线图，支持平滑曲线（Smooth）和数据区域缩放（DataZoom）。

    数据归一化：支持“绝对评论数”与“百分比热度值”切换。

C. 用户画像对比（Demographics）

    等级分布：对比不同视频的 Lv0-Lv6 用户占比（堆叠柱状图或并排饼图）。

    地理分布：基于 location 字段的中国地图热力对比。

D. 语义与内容对比（Content Insights）

    词云对比：并排展示多个视频的高频词云。

    争议性分析：基于 replies/likes 比例，识别争议性根评论。

4. 前端布局规划 (Google Trends Style)

    顶部：横向伸缩的 BV 号对比输入框。

    主体 (Grid 布局)：

        Row 1 (100% width)：时间趋势折线图（主视觉）。

        Row 2 (50% + 50% width)：左侧为地理分布图，右侧为词云展示区。

        Row 3 (100% width)：用户等级构成堆叠图。

5. 待优化的逻辑处理（Agent 需要注意）

    数据清洗：分词需过滤 B 站表情符（如 [热词系列_xxx]）及停用词。

    数据联动：在时间轴上缩放选定时，下方地理和词云图表应支持同步更新（可选）。

你可以这样对 Agent 发指令：

    “请根据以上需求，首先帮我构建 前端 HTML/CSS 框架。要求使用 CSS Grid 实现一个 1600px 宽的看板，包含顶部的 BV 号动态对比栏，以及下方三个维度的图表卡片位。请确保视觉风格接近 Google Trends，简洁且专业。”
